== Overview

All messages follow the format: HTTP-[identifier]: [message]

Module: `de.cuioss.http`

LogMessages Classes:

- `de.cuioss.http.client.HttpLogMessages` - HTTP client operations (HTTP-106–115, HTTP-200–201)
- `de.cuioss.http.forwarded.ForwardedLogMessages` - Forwarded / reverse-proxy header resolution (HTTP-120–123)

== HttpLogMessages

=== WARN Level (100-199)

[cols="1,2,3,3", options="header"]
|===
|ID |Component |Message |Description

|HTTP-106
|Handler
|IO error while pinging URI %s: %s
|Logged when IOException occurs during HTTP ping operation. Parameters: URI being pinged, error message.

|HTTP-107
|Handler
|Interrupted while pinging URI %s: %s
|Logged when thread is interrupted during HTTP ping operation. Parameters: URI being pinged, error message. Note: Not tested due to complexity of reliably triggering InterruptedException.

|HTTP-108
|Adapter
|Failed to serialize request body for %s request: %s
|Logged when serializing a request body via the configured request converter fails. Parameters: HTTP method, error message.

|HTTP-110
|Retry
|Skipping retry for non-idempotent method: %s (idempotentOnly=true)
|Logged when retry is skipped because the HTTP method is not idempotent and idempotentOnly is enabled. Parameters: HTTP method name.

|HTTP-111
|Retry
|%s request failed after %s attempts
|Logged when all retry attempts are exhausted. Parameters: HTTP method, maximum attempts.

|HTTP-112
|Retry
|%s request failed on attempt %s, retrying after %sms
|Logged when a request fails and a retry is scheduled with backoff. Parameters: HTTP method, failed attempt number, delay in milliseconds.

|HTTP-113
|Adapter
|Response conversion failed for status %s
|Logged when converting HTTP response content fails. Parameters: HTTP status code.

|HTTP-114
|Adapter
|Network error during %s request: %s
|Logged when a network error occurs during an HTTP request. Parameters: HTTP method, error message.

|HTTP-115
|Handler
|Using cleartext HTTP for %s (allowInsecureHttp=true); data is transmitted unencrypted
|Logged when a cleartext HTTP handler is built under the allowInsecureHttp opt-in. Parameters: URI.
|===

=== ERROR Level (200-299)

[cols="1,2,3,3", options="header"]
|===
|ID |Component |Message |Description

|HTTP-200
|Adapter
|Configuration error during %s request: %s
|Logged when a configuration error prevents an HTTP request from being executed. Parameters: HTTP method, error message.

|HTTP-201
|Adapter
|Failed to build HTTP request for %s: %s
|Logged when building an HTTP request fails. Parameters: HTTP method, error message.
|===

== ForwardedLogMessages

=== WARN Level (100-199)

[cols="1,2,3,3", options="header"]
|===
|ID |Component |Message |Description

|HTTP-120
|Forwarded
|Rejecting proxy context path with control characters: %s
|Logged when a proxy-supplied context path contains control characters and is rejected. Parameters: sanitized context path.

|HTTP-121
|Forwarded
|Rejecting proxy context path to prevent protocol-relative URL injection: %s
|Logged when a proxy-supplied context path would form a protocol-relative URL and is rejected. Parameters: sanitized context path.

|HTTP-122
|Forwarded
|Rejecting forwarded header %s: value failed security sanitization: %s
|Logged when a forwarded header value fails security sanitization and is rejected. Parameters: header name, sanitized header value.

|HTTP-123
|Forwarded
|Ignoring client IP: forwarded chain carries an unparseable entry: %s
|Logged when the forwarded chain contains an unparseable entry and client IP resolution is aborted. Parameters: sanitized chain entry.
|===

== Notes

=== Test Coverage

All WARN-level LogRecords have test coverage except for:

- HTTP-107 (HTTP_PING_INTERRUPTED) - InterruptedException scenarios

This is documented in the LogRecord Javadoc but not tested due to the complexity of reliably triggering InterruptedException in unit tests.
